# backend/ — 도달성 엔진 (B) + 추천 조합 (C)

프론트엔드가 호출하는 핵심 로직. **순수 함수**로 시작하고, 필요하면 FastAPI로 감싼다.
> ⚠️ 아직 코드 미작성 — v1.0(도달성)부터 단계적으로 구현.

## (B) 도달성 엔진 — 결정론적, NN 아님

```
통과 조건:  OD[현재][후보] + 체류[후보] ≤ 남은예산
```

- `od_matrix.parquet` + 후보 체류시간으로 **하드 필터**.
- "1시간" = 이동시간 + 체류시간 누적 (D5).
- A→B→C 진행마다 예산 차감, **예산 소진까지 반복**(가변 깊이, D6).
- 후보가 인접 구로 넘어가면 그 사실을 결과에 표시(D7).

## (C) 점수·추천 — 통과 후보 정렬

```
점수 = 학습랭커(궤적,후보)          # models/ 의 NN
     ⊕ 사용자 가중 신호(슬라이더)    # 언급량·군집·생활인구  (v1.2)
     ⊕ 성·연령 인기도 조건화(집계)   # (v1.3)
```
