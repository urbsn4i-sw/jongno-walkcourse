# 종로 도보 동선 추천 (가칭: Jongno Walk-Course Recommender) — 프로젝트 핸드오프 문서 **v3**

> **이 문서의 목적**
> 채팅(claude.ai 상담 대화)이 길어져 새 창에서 다시 시작할 때, **이 문서 전체를 새 채팅에 붙여넣으면 맥락이 그대로 복원**되도록 만든 단일 진실 소스(single source of truth).
> 새 채팅 첫 메시지 예시: *"아래는 진행 중인 학교 과제의 핸드오프 문서야. 이걸 기준으로 이어가자. 현재 상태는 13번 섹션을 봐줘."* + (이 문서 붙여넣기)
>
> **참고:** Claude Code(데스크톱 Code 탭)는 새 채팅이 불필요 — 기존 `jongno-walkcourse` 세션 그대로 이어감. "새 세션"은 이 claude.ai 상담 대화에만 해당.
>
> **v3 변경 요지:** **v1.0 데이터 레이어 + 도달성 엔진 + folium 검증 지도 완성**(데이터 파이프라인 7단계 전부 작동). 앱 UX 요구사항 구체화(일반지도·출발점+상단 시간버튼·A→B→C 클릭 루프·undo·카테고리). 랜덤포레스트=후보 랭킹용(경로 아님, NN 베이스라인)으로 명확화. 실제 산출물·노드 수(946)·환경 갱신. 다음 단계 = React+Leaflet 본 구현(v1.1).

## 1. 프로젝트 한 줄 정의
**종로구 지도 위에서, 내 출발점으로부터 도보 예산(30분/1h/2h/3h) 안에 닿는 스팟(맛집·관광지·핫스팟)을 우선순위로 추천하고, 하나를 고르면 다음 후보가 다시 떠서 A→B→C 동선이 완성되는 무료 웹앱.**

- 핵심 상호작용: `A 선택 → B-1…B-n 우선순위 표시 → B-5 선택 → C-5-1…C-5-n …` (예산 소진까지, 가변 깊이)
- **학습 목표(중요):** NN을 *직접 짜고 학습시키는* 경험이 이 과제의 중심. 추천·랭킹 = NN/ML, 시간/도달성 = 결정론.
- 학술 프레이밍(선택): 보행 접근성(walkability) · 시간지리학의 time-space prism

## 2. 절대 제약 (Hard Constraints)
1. **전부 무료.** 유료 API·유료 빌링 금지. 무료 등록(키 발급)은 허용. (Claude Code는 **Claude Max 플랜 포함** → 제약 위반 아님.)
2. **배움 중심.** 결과물보다 학습 경험·재현 가능성 우선. 특히 **NN을 적극적으로 구축·학습**.
3. **지도에 경로와 추천이 반드시 보여야 함.** (텍스트 리스트만으로는 불가)
4. **경계 침범 허용·안내.** 도보 예산이 크면 종로구를 벗어나 **중구·서대문·성북·은평**으로 넘어갈 수 있음 → 데이터·네트워크가 버퍼 포함, UI는 다른 구로 넘어감을 표시.
5. 식당 폐점 여부 검증은 **사용자가 직접 처리**(앱 범위 밖).

## 4. 아키텍처 (4 레이어) — A·B·D 1차 완성
```
(A) 데이터 레이어   : 종로+버퍼 POI 테이블(피처) + 도보 OD행렬   [✅ 완성]
(B) 도달성 엔진     : OD[현재][후보]+체류[후보] ≤ 남은예산 인 후보만 통과  [✅ 완성, 결정론적]
(C) 점수·추천 레이어 : 통과 후보 정렬  ← ★ NN/ML이 사는 곳 (현재 hotspot_score 순 임시)
        점수 = 학습랭커 ⊕ 사용자가중신호 ⊕ 성·연령 인기도조건화
               ↑ NN/랜덤포레스트(직접 구축·학습)   ↑ 언급량·군집(슬라이더)  ↑ 집계
(D) 화면(웹)        : 지도 + 컨트롤(시간버튼·설정) + 우선순위 마커 + 동선 + 클릭 루프
                      [folium 검증 지도 ✅ / React 본 구현 ⬜ = 다음 단계]
```
**원칙:** 하드 제약(시간 예산)은 (B)가 정확히 거르고, 그 안에서 (C)의 NN/ML이 순위만 매긴다.

## 5. ★ 앱 UX 요구사항 (사용자 확정 — React 구현 대상)
> v1.1 React 본 구현에서 만들 화면 동작. 백엔드 엔진(reachability)은 이미 작동하므로, 클릭 UI를 입히는 작업.

1. **지도:** 네이버지도 느낌의 **깔끔한 일반지도**(무료 타일로 근사). (D16)
2. **초기 화면:** **출발점(최초 지점)만** 표시. 화면 **상단에 시간 버튼(30분/1h/2h/3h) + "설정" 버튼**.
3. **시간 선택 + 설정 클릭 시:**
   - 출발점에서 **가장 가까운/추천 A 후보들(A-1, A-2, … A-5…)** 제시
   - 사용자가 하나 선택(예: A-2) → **그 지점 기준 B 후보(B-2-1, B-2-2, …)** 제시
   - 같은 식으로 **C, D … 예산 소진까지** (가변 깊이)
   - **30분이면 적게, 3시간이면 많이** 제시됨 (예산이 깊이·후보 수 결정)
4. **뒤로 가기(undo):** 잘못 선택 시 이전 단계로 — **예산 복구 + 후보 재표시**(스택 패턴)
5. **카테고리 선택(식사/관광/머물공간 = 음식점/관광명소/카페):** 각 단계에서 필터 (데이터에 이미 있음)
6. **후보 랭킹:** 현재 hotspot_score 순 → 나중에 **랜덤포레스트/NN 랭커로 교체**(D18, C 레이어)

## 6. ★ NN/ML 사다리 (이 과제의 핵심 — 직접 구축·학습)
> **방침:** Foursquare 등 공개 체크인으로 *방법을 학습·검증*(정량지표), 구조를 종로 POI에 적용(데모). 각 레벨에서 **모델 정의 → 학습 루프 → loss 관찰 → 평가**를 직접 손으로.

| Level | 모델 | 직접 경험 | 산출/근거 |
|---|---|---|---|
| **베이스라인** | **랜덤포레스트**·인기순·KNN·로지스틱·결정트리·SVM | 고전 ML로 후보 랭킹, NN과 비교 기준 | Recall@k·NDCG 비교군 |
| **L1** | MLP 랭커 (`nn.Linear`+ReLU → 점수) | 모델 정의·순전파·손실·옵티마이저·학습 루프 | NN 입문 완주 |
| **L2** | 학습된 임베딩 (`nn.Embedding`) | 표현 학습, UMAP 시각화 | POI가 종류별로 뭉치는 그림 |
| **L3** | 시퀀스 모델 (LSTM / 소형 Transformer) | A→B→C를 *순서*로 인코딩, 다음 장소 예측 | "층층이 펼쳐지는 NN" |
| **L4** *(선택)* | GNN (POI 그래프 메시지 패싱) | 그래프 표현학습 | 고급 확장 |
| **v2** *(연기)* | 강화학습 / 확산(trajectory) | 동선 전체 보상 최적화 / 동선 생성 | future work |

**과학적 서사:** 랜덤포레스트 등 베이스라인 vs NN을 **Recall@k·NDCG로 대결** → "고전 대비 NN 우위"를 수치로 증명. (랜덤포레스트는 D18대로 **후보 랭킹**에 쓰는 것 — 경로 잇기 아님.)

## 8. 데이터 파이프라인 — ✅ 완성 (실제 산출물)
**7단계 스크립트, 모두 커밋·푸시 완료. 데이터는 `.gitignore`(코드만 레포에).**

**핵심 산출물 (data/processed/, gitignore):**
- `pois_final.parquet` — **946개 POI** (피처: category_main, gu, lon, lat, mention_count, hotspot_score, source 등)
- `od_matrix.parquet` — **946×946 도보시간(분)**, 행/열=place_id
- `walk_graph.gpickle` — 보행망(경로 폴리라인용)
- `node_snap.parquet` — POI↔도로망 매핑

### 핫스팟 점수 (전처리, NN 아님)
```
hotspot_score = 0.4·z(log(언급량), 99%윈저라이즈, 일반어가드) + 0.6·(반경100m 이웃수 밀집도)  → 0~1
DBSCAN cluster_id는 핫스팟 권역 표시용으로 보존
```

### 데이터 품질 메모(보고서 자산)
- 네이버 언급량 1차 수집 시 **일반명사 상호("집"·"안녕"·"함께있어좋은사람")가 무관한 글을 대량 매칭 → 거짓 폭증**(최대 2,100만) 발견 → 검색어 `"이름" 구`(정확구+구) + 윈저라이즈 + STOPWORD 가드로 해결. (v1 백업 `pois_mentions_v1.parquet` 보존 — 개선 전후 비교 자료)
- 핫스팟 상위 검증: 젊음의거리(0.977)·익선동한옥거리(0.927) 등 실제 핫플이 상위 = 점수 타당.

## 9. 도달성 엔진 (레이어 B) — ✅ 완성
`backend/reachability.py` (+ `test_reachability.py`):
```python
reachable(current_place_id, remaining_budget_min, category_filter=None, visited=None)
# 통과: OD[현재][후보] + 체류시간[후보] ≤ 남은예산
# 체류시간: 음식점 40 / 카페 20 / 관광명소 30분 (STAY_MINUTES 상수)
# 제외: NaN(도달불가)·자기자신·visited / 정렬: hotspot_score 내림차순(나중에 NN이 대체)
# 반환: place_id,name,category_main,gu,lon,lat,od_min,stay_min,remaining_min(방문 후 잔여),hotspot_score
```
**검증 통과:** 경복궁 60분 → 467곳 도달(경계값 정확). 카테고리 필터(식사 151/관광 158/머물 158=467 일치). A→B→C→D 체이닝(경복궁 120분 → 젊음의거리 → 해운대, 잔여예산 누적·visited 제외 정상). 엣지(10/5/0분 빈 결과). **`remaining_min`이 다음 단계 예산으로 그대로 넘어가 체이닝 구동.**
