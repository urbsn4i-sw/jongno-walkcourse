# 종로 도보 동선 추천 (가칭: Jongno Walk-Course Recommender) — 프로젝트 핸드오프 문서

> **이 문서의 목적**
> 채팅이 길어지거나 이미지 수 제한으로 새 창에서 다시 시작해야 할 때, **이 문서 전체를 새 채팅에 붙여넣으면 맥락이 그대로 복원**되도록 만든 단일 진실 소스(single source of truth)입니다.
> 새 채팅 첫 메시지 예시: *"아래는 진행 중인 학교 과제의 핸드오프 문서야. 이걸 기준으로 이어가자. 현재 상태는 11번 섹션을 봐줘."* + (이 문서 붙여넣기)

---

## 1. 프로젝트 한 줄 정의
**종로구 지도 위에서, 내 출발점으로부터 도보 예산(30분/1h/2h/3h) 안에 닿는 스팟(맛집·관광지·핫스팟)을 우선순위로 추천하고, 하나를 고르면 다음 후보가 다시 떠서 A→B→C 동선이 완성되는 무료 웹앱.**

- 핵심 상호작용: `A 선택 → B-1…B-n 우선순위 표시 → B-11 선택 → C-1…C-n …` (예산 소진까지)
- 학술 프레이밍(선택): 보행 접근성(walkability) · 시간지리학의 time-space prism

---

## 2. 절대 제약 (Hard Constraints)
1. **전부 무료.** 유료 API·유료 빌링 금지. 무료 등록(키 발급)은 허용.
2. **배움 중심.** 결과물보다 학습 경험·재현 가능성 우선. 오픈소스 도구 선호.
3. **지도에 경로와 추천이 반드시 보여야 함.** (텍스트 리스트만으로는 불가)
4. **경계 침범 허용·안내.** 도보 예산이 크면 종로구를 벗어나 **중구·서대문구·성북구·은평구**(및 경계상 동대문구·강북구 일부)로 넘어갈 수 있음 → 데이터·네트워크가 이 버퍼를 포함해야 하고, UI는 다른 구로 넘어감을 표시.
5. 식당 폐점 여부 검증은 **사용자가 직접 처리**(앱 범위 밖).

## 4. 아키텍처 (4 레이어)
```
(A) 데이터 레이어   : 종로+버퍼 POI 테이블(피처) + 도보 OD행렬   [오프라인 사전계산]
(B) 도달성 엔진     : OD[현재][후보]+체류[후보] ≤ 남은예산 인 후보만 통과  [결정론적, NN 아님]
(C) 점수·추천 레이어 : 통과 후보 정렬
        점수 = 학습랭커(궤적,후보) ⊕ 사용자가중신호 ⊕ 성·연령 인기도조건화
               ↑ NN(Foursquare 학습)   ↑ 언급량·군집·생활인구(슬라이더)  ↑ 집계
(D) 화면(웹)        : 지도 + 컨트롤(예산·신호 슬라이더·인구 필터) + 우선순위 마커 + 동선 폴리라인
```
**원칙:** 하드 제약(시간 예산)은 (B)가 정확히 거르고, 그 안에서 (C)의 NN이 순위만 매긴다.

## 6. 데이터 수집 범위 & 파이프라인
- **공간 범위:** 종로구 경계 + 약 2~4km 버퍼(3시간 도보 ≈ 13km 대비). 버퍼가 중·서대문·성북·은평을 포함해야 경계 침범 동선이 작동.
- **노드 캡:** v1은 ~500~800개로 제한(카테고리별 기초 relevance 상위)해 OD행렬을 다룰 만하게.
- **파이프라인:** bbox/구경계 설정 → POI 수집(중복제거) → 피처 보강(언급량·생활인구·핫스팟 점수) → 테이블(parquet) 저장 → OSMnx로 보행망 받기 → 노드 간 OD행렬 사전계산 → 저장.
- 이 수집 파이프라인 자체가 GitHub 산출물.

### 핫스팟 점수 (전처리, NN 아님)
```
hotspot_score = z(블로그/검색 언급량) + DBSCAN 군집 밀집도
              → 0~1 정규화 후 노드 피처로 추가
```

## 7. ML 구성 (랭커만)
- **방법 검증(신뢰성):** Foursquare 공개 체크인으로 next-POI 랭커 학습·평가, 정량지표 **Recall@k / NDCG** 보고. ← "진짜 ML 했다"의 물증.
- **베이스라인 → 딥러닝 서사:** KNN·로지스틱·의사결정나무·SVM·앙상블을 베이스라인으로, RNN/Transformer/GNN과 비교.
- **표현 학습:** 군집(DBSCAN), 차원축소(PCA/UMAP)로 POI 임베딩.
- **서울 적용(데모):** 학습된 구조를 종로 POI에 적용. 전이 신호로 데이터랩 연관관광지 + 부족분은 합성 궤적.
- **정직한 한계:** Foursquare→종로 도메인 갭 존재. 사용자에게 보이는 순위는 해석가능 신호(언급량·군집·연관관광지)가 주로 끌고, 학습랭커는 벤치마크 증거 역할 겸함.

## 9. 빌드 단계 (각 단계가 독립 작동·데모 가능)
```
v1.0  데이터+OD행렬+도달성+지도(단일 스텝)        → "도보 n분 스팟지도" 완성
v1.1  A→B→C 반복 루프 + 누적예산 + 동선 폴리라인  → 핵심 UX 완성
v1.2  사용자 신호 가중 추천(슬라이더)              → "우선순위 추천" 완성
v1.3  성·연령 인기도 조건화                        → 개인화 맛보기
v1.4  Foursquare 학습 랭커 + 벤치마크              → "진짜 ML" 증거
```
핵심 상호작용은 **v1.1에서 이미 작동** → 뒤 단계가 늦어도 보여줄 게 있음.
